# Change Log

All notable changes to the "Baseline Theme" extension will be documented in this file.

## [1.0.2] - 2024-12-25

### Fixed
- Aligned light theme color scheme to exactly match dark theme
- Fixed docstring quotes color in light theme
- Fixed decorator color in light theme
- Fixed module and submodule access colors in light theme
- Fixed assignment operator color in light theme
- Fixed type hints color in light theme
- Fixed format placeholder color in light theme
- Fixed function call color in light theme
- Fixed highlight background transparency issues
- Removed unsupported background property from Markdown Code token rule
- Added TypeScript styling to match Python syntax highlighting

### Changed
- Light theme now uses identical color scheme as dark theme for all syntax elements
- Improved consistency across all token types between dark and light variants

## [1.0.1] - 2024-12-22

### Fixed
- Removed broken image reference in README

## [1.0.0] - 2024-12-22

### Added
- Initial release
- Dark theme variant
- Light theme variant
- Optimized Python syntax highlighting
- Function definition names styled in grey
- Semantic highlighting support
- Based on Obsidian Baseline theme by Alexis C

### Features
- Clean, minimal interface colors
- Carefully balanced syntax colors for readability
- Support for VS Code, Cursor, and Kiro editors
